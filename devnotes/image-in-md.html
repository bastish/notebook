In Astro, you can add images to your Markdown files by placing the images in either the public/ directory or the src/assets/ directory, depending on your preference and needs. Here’s how:

Option 1: Place Images in the public/ Directory

The public/ directory is for static files that are served as-is. When you put images here, you can reference them directly by their URL.

  1. Place your image in the public/ directory: For example: public/images/campground.jpg

  2. Link to the image in your Markdown:

    # My Trip

Here's the campground:

![Campground](images/campground.jpg)

  3. Output: Astro will render this as:

    <img src="/images/campground.jpg" alt="Campground">

Option 2: Place Images in the src/assets/ Directory

If you use the src/assets/ directory, Astro will process and optimize these images during build time, allowing you to use image transformations (e.g., resizing).

  1. Place your image in the src/assets/ directory: For example: src/assets/images/campground.jpg

  2. Update the Markdown to include the image: Use Astro’s Image component if you want optimized images:

    # My Trip

Here's the campground:

![Campground](images/campground.jpg)

  3. Output: You may need to adjust the frontmatter or use custom logic to handle the image. For most use cases, Astro’s build process will manage it effectively.

Key Differences

Featurepublic/ Directorysrc/assets/ Directory
Image URLStatic and directly usableNeeds to be processed or referenced via Astro
Build OptimizationNo optimizationOptimized during build time
Use CasesExternal/static resourcesImages requiring transformations

Recommendation

  • Use the public/ directory for simple, static image hosting.
  • Use src/assets/ for optimized or dynamically processed images.
